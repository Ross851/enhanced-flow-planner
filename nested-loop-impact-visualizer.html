🚀 How to Fix This Disaster

1. Use OData Filtering at Source
Instead of getting all items then looping, use $filter in Get Items: "Status eq 'Active'" reduces 1000 items to 50 items = 95% reduction!
2. Replace Nested Loops with Joins
Use Select with joins or Filter Array instead of nested Apply to Each. Single operation instead of N×M operations.
3. Batch Operations
Use batch APIs: SharePoint batch, Graph API batch. Process 100 items in 1 call instead of 100 calls.
4. Implement Pagination
Process in chunks with Do Until: 100 items at a time with delays. Prevents throttling while handling large datasets.
5. Use Select and Expand
Get related data in one call: $expand=Department instead of separate lookups. Eliminates need for nested loops entirely.

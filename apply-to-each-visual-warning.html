Critical Optimisation Strategies

1. ALWAYS Filter at Source
Use OData filters in Get Items: Status eq 'Active' and Modified gt '2024-01-01' This can reduce iterations by 90-95%!
2. Use Select Before Apply to Each
Map complex objects to simple structure. Only include fields you need. Reduces memory usage and improves performance by 50%.
3. Enable Concurrency (Carefully)
Set to 20-50 for parallel processing. Watch API limits! Can improve speed by 10x but increases API calls per minute.
4. NEVER Nest Apply to Each
100 outer √ó 50 inner = 5,000 iterations! Use Filter Array or restructure data. Exponential complexity will cause failures.
5. Consider Batch Operations
Instead of updating items one by one, use SharePoint $batch or SQL bulk operations. Reduces 1000 API calls to 1!
